This tutorial will show you how to get your own instance of Deepfloyd's IF model running on Cerebrium. 
Using the custom python script deployer you'll be able to get your model to do exactly what you need.

To see the final implementation you can view it [here](https://github.com/CerebriumAI/examples/tree/master/11-deepfloyd-text-image-model)


## Basic Setup

It is important to think of the way you develop models using Cerebrium should be identical to developing on a virtual machine or Google Colab.

To start we need to create a **main.py** file which will contain our main Python code. 
For our purposes today the code is simple, so we can do everything in 1 file.

Other than the **main.py** file, we will also need to declare what libraries (**requirements.txt**) as well as packages (**pkglist.txt**) that we will need.

We can find the libraries that we need in the [Hugging face documentation](https://huggingface.co/DeepFloyd/IF-I-XL-v1.0) for DeepFloyd's IF model.
Reading their doccumentation, we see that we need the following libraries:
```
diffusers
accelerate
transformers
safetensors
sentencepiece
torch
```
So let's add these entries to **requirements.txt** as shown above.

Now that our libraries have been installed, we can start working on our **main.py**.  
The first step is to get all our imports and define the model as follows:

```python
import base64
from io import BytesIO
from diffusers import DiffusionPipeline
from diffusers.utils import pt_to_pil
import torch
import requests
from pydantic import BaseModel


########################################################################
# Run once on first deployment to initialise the model
########################################################################
access_token = '<<<Insert your API access token here>>>'

# stage 1
stage_1 = DiffusionPipeline.from_pretrained("DeepFloyd/IF-I-XL-v1.0", variant="fp16",
                                             torch_dtype=torch.float16, use_auth_token=access_token) # Private model requires API Key.
stage1 = stage_1.to("cuda")     # So we can leverage our gpu deployment


# stage 2
stage_2 = DiffusionPipeline.from_pretrained("DeepFloyd/IF-II-L-v1.0", text_encoder=None,
                                             variant="fp16", torch_dtype=torch.float16, use_auth_token=access_token)
stage2 = stage_2.to("cuda")

# stage 3
safety_modules = {"feature_extractor": stage_1.feature_extractor, "safety_checker": stage_1.safety_checker, "watermarker": stage_1.watermarker}
stage_3 = DiffusionPipeline.from_pretrained("stabilityai/stable-diffusion-x4-upscaler",
                                             **safety_modules, torch_dtype=torch.float16, use_auth_token=access_token)
stage3 = stage_3.to("cuda")

```

We define this code to declare the model at the top of the file for two reasons:

1. Code loaded in the top of the file is automatically run when deployed, meaning the model can be downloaded and loaded into memory on the first deployment.
   The first deployment is the longest but thereafter it should be pretty quick.
2. If this was the predict function, it would not be called when we first deploy but instead would only be called on the first request. Additionally, it would be loaded from cache on every request. Both cases mean our model will be slow to return a prediction.

Since the DeepFloyd IF model is a private model, we need to include our API key as an authorisation token. This will tell HuggingFace who you are and show that you have permission to access the model.

Now we've set everything up and can write our **predict** function which is called whenever the user makes a request. The predict function received 3 parameters:

1. The request object you defined
2. The run_id which is a unique identifier for each inference request
3. A logger object which you should use to log any contents. You should use logger.info(...) or logger.error(...)

<!-- Introduce prediction function -->


```python

########################################################################
# Run every time your model is called.
########################################################################

def predict(item, run_id, logger):
    item = Item(**item)
    ##Do something with parameters from item
    # Retrieve text embeddings
    prompt_embeds, negative_embeds = stage_1.encode_prompt(Item.prompt)

    # stages = []
    logger.info(f'Starting with prompt: {Item.prompt}')

    # Running stage 1
    generator = torch.manual_seed(0)
    image = stage_1(prompt_embeds=prompt_embeds, negative_prompt_embeds=negative_embeds,
                     generator=generator, output_type="pt").images
    # stages.append(pt_to_pil(image)[0])
    logger.info('Run stage 1')

    # Running stage 2
    image = stage_2(image=image, prompt_embeds=prompt_embeds, negative_prompt_embeds=negative_embeds,
                    generator=generator, output_type="pt").images
    # stages.append(pt_to_pil(image)[0])
    logger.info('Run stage 2')

    # Running stage 3
    image = stage_3(prompt=Item.prompt, image=image, generator=generator, noise_level=100).images
    # stages.append(image[0])
    logger.info('Run stage 3')

    # Converting and returning the finished image.
    logger.info('Converting')
    buffered = BytesIO()
    image[0].save(buffered, format="PNG")
    finished_image = base64.b64encode(buffered.getvalue()).decode("utf-8")

    logger.info('Returning image')
    return finished_image
```
