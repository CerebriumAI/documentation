---
title: "REST API"
description: "Make authenticated HTTP requests to your Cerebrium endpoints"
---

By default, all functions on Cerebrium are accessible via authenticated POST requests, unless they are marked as private by prefixing the function name with an underscore (for example `_private_function()`). Authentication is handled using the JWT token available in the **API Keys** section of your dashboard.

## Request format

The POST request follows the structure below, where `{function}` is the name of the function you want to invoke. For example, in this case, the function `predict()` from `main.py` is being called.

```bash
curl --location --request POST 'https://api.aws.us-east-1.cerebrium.ai/v4/p-xxxxx/{app-name}/{function}' \
--header 'Authorization: Bearer <JWT_TOKEN>' \
--header 'Content-Type: application/json' \
--data '{
  "function_param": "data"
}'
```

## Response format

Responses follow this format:

```json
{
  "run_id": "52eda406-b81b-43f5-8deb-fcf80dfsb74b",
  "run_time_ms": 326.34,
  "result": { "some": "data" }
}
```

## Status codes

- `200` → the function completed successfully
- `401` / `403` → authentication or authorization failed
- `404` → the app or function path could not be found
- `5xx` → either your app raised an exception or the platform could not complete the request

If you want to return a custom status code such as `422` or `404`, include a `status_code` field in the JSON response from your `main.py`.

<Note>
  If you are debugging a failed request, check both the HTTP status code and the
  app logs in the Cerebrium dashboard. Most production debugging goes faster
  when you inspect the request payload, logs, and `run_id` together.
</Note>
